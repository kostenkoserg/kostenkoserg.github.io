Развернуть свой персональный блог, блог компании или лендинг проекта используя [github](https://pages.github.com/) - задача не сложная и не требующая особых навыков. С другой стороны от необходимости чтения документации и мук выбора того или иного решения имеют место быть. Я, максимально констуктивно, постараюсь изложить основные шаги, которые необходимо сделать для того, что бы обзавестись личным блогом на небезизвестном ресурсе.

#### Шаг 1. Хостим статику на github

Для этого нам потребуются аккаунт на [github](https://github.com/) и базовые навыки работы с [git](https://git-scm.com/). К слову сказать, все ниже изложенное можно проделать и для конкурирующего сервиса [gitlab](https://gitlab.com/), - где кому больше нравится.

После чего, нужно просто создать проект, назвав его особым образом, а именно <логин>.github.io, например kostenkoserg.github.io (см. скрин)

![github pages project screenshot](/img/my_github_srcshot.png)

В принципе, на этом и вся подготовительная рабобта для хостинга статики завершена. Далее проосто клонируем проект себе (пока пустой), создаём index.html пушим в репозиторий:

```
git clone https://github.com/kostenkoserg/kostenkoserg.github.io.git
cd kostenkoserg.github.io
echo "Hello GitHub!"
git commit -m "my first github page"
git push
```
Все! Страничка доступна по https://kostenkoserg.github.io

![first_github_page](/img/first_github_page.png)
По желанию, можно подвязать уже существующий домен, если по каких-либо соображениям .github.io не подходит.
